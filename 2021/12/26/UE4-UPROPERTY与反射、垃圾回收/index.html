UE4 UPROPERTY与反射、垃圾回收

为了搞清楚加不加 UPROPERTY有什么区别，查阅了很多和 UPROPERTY有关的资料，官方文档和社区维基提到跟反射以及垃圾回收有关系，而很多教学视频和文章都只是在介绍UPROPERTY的说明符怎么用……

反射

反射是指计算机程序在运行时可以访问、检测和修改它本身状态或行为的一种能力。

UE4使用其自己的反射实现来支持动态功能，如垃圾回收、序列化、网络复制和蓝图/C++通信。这些功能是可选的，意味着您必须将正确的标记添加到类型，否则虚幻将忽略它们，而不会为它们生成反射数据。

垃圾回收

在UE4垃圾回收程序中，有一个概念叫做根集。该根集基本上是一个对象列表，这些对象是回收程序知道将不会被垃圾回收的对象。只要根集中的某个对象到一个对象存在引用路径，就不会对所涉及对象进行垃圾回收。如果某个对象不存在到根集的此类路径，则称为无法访问，将会在下次运行垃圾回收程序时将其回收（删除）。引擎按特定的时间间隔运行垃圾回收程序。

UPROPERTY

标明修饰的变量将加入反射系统。

  • UObject* 类型
    • 非 Actor 和 ActorComponent：得加 UPROPERTY，不然它不是根集的一部分，最终，垃圾回收程序会检测到该对象无法访问，从而将其销毁。那么这个指针就变成了空悬指针，指向已经释放的内存，而你还不知道，之后使用该指针时就可能错误地把后来存入这块内存的数据当作是你想要的
    • Actor 和 ActorComponent：也得 UPROPERTY，只不过有点不一样，因为 Actor 通常被链接回到根集的 Object 引用（例如它们所属的关卡），而 ActorComponent 被 Actor 自身引用，也就是说它们直接就是链接到根集上，不需要 UPROPERTY 干这一步，但仍需要 UPROPERTY 在它们被销毁时做一项工作，那就是将引用它们的指针置为空指针，这一点上是其他 UObject 类型是一致的
  • 普通类型
    • 非容器：可以不加 UPROPERTY，只要不需要动态功能。比如定义类成员变量 int count，不需要序列化，不需要网络复制，不需要在蓝图上编辑值（让引擎自动设置为 0 或者类内初始化 int count = 10），就只在类内部使用
    • 容器：
      • 存储的是普通类型：和非容器一致
      • 存储的是 UObject* ：需要加 UPROPERTY，理由和单独的 UObject* 一致。使用 UPROPERTY 修饰容器相当于用 UPROPERTY 修饰容器中的每个元素

所以基本上都需要使用 UPROPERTY ,如果不清楚的话，用就对了。
